Functional Description


Component Goals


The goal of the component is to cleanse and validate Czech phone numbers.

The component validates all the czech numbers including non-regional and special numbers like VOIP, shared-prize, audiotex (9 digit numbers). Does not include emergency or non-standard contact numbers e.g. 112, 150, 155, 156, 158, 1188, 116111, etc.

Interface

  • Input: phone number
  • Output: standardized and best existing value of input phone number, phone extension number, scoring and explaination of all discrepancies in the data

Example of standardized phone number: +420777111222

Component Behaviour


Component replaces synonyms for "extension" (e.g. "ext.", "klapka") with snowman character, which is afterwards used as separator between phone number and optional extension number.
Input string can be parsed using following patterns:
optional comment, optional exit code ("00" or "+"), six-fifteen-digit long international phone number, snowman character, one-five-digit long extension, optional comment
optional comment, optional exit code ("00" or "+"), six-fifteen-digit long international phone number, optional comment

Parsed phone numbers are cleansed. They are considered to be Czech numbers in cases when they have correct IDC (international dialing code) = "420" or have maximally 9 digits (10 if they are in old format and begin with zero).
Phone numbers in old format (before year 2001) are converted into the new one. After that, phone prefixes are validated against mobile and landline lookups.

Phones with exit code but without Czech IDC are validated against lookup with all international IDCs to see, whether they might be valid.

Special prefixes e.g. list of non-regional and special numbers like VOIP, shared-prize, audiotex, etc. will be recognized as one special group and get the same explanation code.

Lookup Files and External Sources


Component uses lookups with extension synonyms, mobile/landline prefixes and prefixes with international IDCs. Besides that, lookup for old format conversion is used.

Summary of Testing Data


Testing data contains set of phone numbers, where some of them are valid mobile/landline numbers, have comments, extension or they are foreign, invalid, unparsed.

Parameters

Parameter name Description
Phone number output format Phone number output format definition. Default value is empty which means that the original component logic will be used. Example of unique format including extension is '+ddd ddddddddd ext. dddddd'. "d" is successively replaced by phone number digits. Other characters will be left as they are. Only OUT_NUMBER for identified values (STD value is available) is standardized in that way.

Attribute Details

sco_number

Data quality score

Value Description
0 Valid value. No modification was done to the input value, it was correct on input.
< 10 000 Corrected value. Small modification was done to the input value (trim, squeeze spaces,removing non-digits) but resulting value is still valid.
< 10 000 000 Unsafe/Unknown value. Huge modification was done to the input value or value does not fit the pattern requirements or reference data (input value does not fit the correct length or pattern or is not found in reference data) and resulting value can no longer be considered valid
>= 10 000 000 Null value. Input value or pre-cleansed value (after deleting invalid characters) is null.

exp_number

Explanation codes

Value Description
PHONE_MOBILE Scored 0. Phone number has mobile prefix
PHONE_LANDLINE Scored 0. Phone number has landline prefix
PHONE_SPECIAL Scored 0. Phone number has special prefix
PHONE_EXTENSION Scored 0. Phone number has extension
PHONE_COMMENTS Scored 1 for every char before/after phone number. Maximally 100
PHONE_OLD_FORMAT_FIXED Scored 10. Old format was converted
PHONE_SPECIAL_CHARS Scored 100 for every special character inside phone number. Maximally 500
PHONE_INVALID_EXTENSION Scored 1000. There was extension synonym, but extension number was missing or invalid (e.g. too long)
PHONE_MISSING_EXIT_CODE Scored 2000. Missing exit code in Czech number in international format (with IDC)
PHONE_MISSING_IDC Scored 5000. There was exit code without Czech IDC
PHONE_PREFIX_NOT_FOUND Scored 10000. Phone prefix was not found among mobile/landline prefixes
PHONE_FOREIGN_IDC Scored 100000. Phone had exit code with existing IDC
PHONE_INVALID_DIGIT_COUNT Scored 200000. Czech phone has invalid digit count
PHONE_UNKNOWN_IDC Scored 300000. Phone had exit code with non-existing IDC
PHONE_UNKNOWN Scored 500000. Phone is longer than 9 digits, hasn't Czech IDC and has no exit code
PHONE_NOT_PARSED Scored 1000000. Phone was not parsed
PHONE_EMPTY Scored 10000000. Input string is empty
