Functional Description

 contents 
Component Goals

The goal of this component is to decline Czech names into 4th and 5th grammatical case (accusative and vocative)


Interface

Input: first name, middle name, last name, gender
Output: first name in accusative, middle name in accusative, last name in accusative, 
             first name in vocative, middle name in vocative, last name in vocative, 
             explanation, score

Input requirements

    

  • First name and middle name can contain only one word (letters only).
    • 

  • Last name can have multiple words separated with space or dash. However, only last word is declined.
    • 

  • Gender and at least one name part are populated.
    • 




Component Behaviour
         
Component cleanses input name parts, checks for unsupported characters. If name part contains unsupported character(s), only this part is removed from declension process.
Supported characters for first and middle name are: letters
Supported characters for last name are: letters, - , space


Declension process

Complex algorithm dealing with individual suffix letters is used in included components:
- cz_name_declension_internal/comp_declension_female.comp	
- cz_name_declension_internal/comp_declension_male.comp	



Lookup files and external sources

Internal components use lookups for oriental female and greek/latin male names, which have some declension specifics.


Summary of testing data

Testing data contains set of input names (first, middle, last names and gender) with different suffixes so various declension patterns are used.
